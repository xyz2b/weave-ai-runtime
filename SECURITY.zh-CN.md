# 安全策略

[English](SECURITY.md) | 简体中文

## 支持版本

本仓库仍在持续开发中。
安全修复预计会直接落在当前主开发线上，而不是同时维护多条发布分支。

## 报告漏洞

对于尚未修复的漏洞，请不要直接公开提 issue。
请改用以下方式：

1. 如果你有维护者的私下联系路径，优先使用它
2. 如果仓库启用了 GitHub Security Advisories，则使用该渠道
3. 附上复现步骤、影响范围，以及你认为可行的缓解建议

## 你可以期待什么

维护者会尽量：

- 确认收到报告
- 复现问题
- 评估影响和修复优先级
- 在修复或缓解方案准备好后协调披露

## 适用范围说明

报告时请说明问题影响的是哪一层：

- runtime core 行为
- package 组合
- host 或 permission 集成
- 仅影响 examples 或开发工具链
