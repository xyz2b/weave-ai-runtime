# Validation Findings

This page indexes the maintainer-facing validation ledgers.

## Who is this for?

- Maintainers and contributors working on this repository rather than first-time framework adopters.

## Prerequisites

- Read `../README.md` first so the public docs flow stays intact.
- Use `../../examples/README.md` as the runnable validation path while changing the repo.

## Demo validation findings

- `demo-validation-findings.md`
  - focused findings tied to the runnable demo path

## Runtime validation findings

- `runtime-validation-findings.md`
  - broader runtime-level validation review and open gaps

## How to use these files

- use `../../examples/README.md` to understand the runnable validation path
- use the ledgers here to track what has been validated and what still needs follow-up
- keep maintainer findings out of the root landing page and basic getting-started flow

## Next step

- Use `../../examples/README.md` to rerun the validation path behind a finding or follow-up item.
- Open `demo-validation-findings.md` or `runtime-validation-findings.md` when you need the detailed ledger entries.
- Return to `../guides/testing-and-observability.md` if you are adding a new validation command or diagnostics story.
