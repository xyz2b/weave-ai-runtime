# 验证结论索引

这个页面索引面向维护者的验证台账。

## 适合谁？

- 正在维护这个仓库、而不是第一次采纳框架的维护者与贡献者。

## 前置条件

- 先读 `../README.md`，确保公开文档路径不被打断
- 在修改仓库时，使用 `../../../examples/README.zh-CN.md` 作为可运行验证路径

## Demo 验证结论

- `demo-validation-findings.md`
  - 与可运行 demo 路径绑定的聚焦发现

## Runtime 验证结论

- `runtime-validation-findings.md`
  - 更广义的 runtime 级验证回顾与开放缺口

## 如何使用这些文件

- 先用 `../../../examples/README.zh-CN.md` 理解可运行验证路径
- 再用这里的台账追踪哪些内容已验证，哪些仍需跟进
- 不要把 maintainer findings 放回根 landing page 或基础 getting-started 流程

## 下一步

- 若想重跑某条 finding 对应的验证路径，回到 `../../../examples/README.zh-CN.md`
- 需要详细台账条目时，打开 `demo-validation-findings.md` 或 `runtime-validation-findings.md`
- 如果你在增加新的验证命令或诊断叙事，回到 `../guides/testing-and-observability.md`
